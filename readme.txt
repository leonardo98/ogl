Very Dangerous OpenGL Engine - experiment. Do Not Use It For Any Purpose!
 
Очень опасный OpenGL движек - эксперимент. НЕ ИСПОДЛЬЗУЙТЕ его ни в каких целях!


 * Концепция движка *
# Формальный запрет на использование new & delete
# Все ресурсы - это Акторы, элементы сцен-графа
# Все акторы сцены создаются и удаляются средствами сцен-графа
# Разделение потоков 
## главный поток отвечает только за обновление положения спрайтов и за вывод их на экран и за получение сигналов от устройств ввода
## игровой поток содержит в себе загрузку ресурсов и игровую логику
# Пользователь движка работает исключительно в игровом потоке
# Акторы должны уметь работать в обоих потоках и корректно обрабатывать обращения к себе
# Конструкторы акторов вызываются только из игрового потока, в них может быть обращение к дискам и другие медленные операции
# В главном потоке вызываются только Render & Update акторов
# В Render & Update акторов нельзя использовать потенциально медленные операции, типа обращений к дискам или lock потока(можно try_lock())
# Все акторы - минимального размера, жесткий запрет на превышение 300 строк кода на актор.


* используемые либы *
glfw / glfm - desktop / modile application main loop (glfm is not used yet)
glew - loading extensions of OpenGL
assimp - loading 3D models
bullet - physic


 * идеи *
# в 2D передавать в шейдер минимальное количество параметров(x, y, u, v, alpha)
# в 3D сделать возможность lowpoly модели красить в цвета по одномерной текстуре (в шейдер пойдет x, y, z, u)
# оптимизировать умножение на матрицу в батчинге
# оптимизировать применение трансформаций, если они не оказывают влияние(нулевое смещение или вращение, единичный скейл)
# в 3D реализовать универсальный Ambient Occlusion для PC и мобилок
# Трекинг по web-камере, распознавание простых предметов(типа мяча), управление жестами
